# UI/UX 競品拆解與新版方向 · 2026-04-25

## 目標

使用者指出第一版 demo 與正式版差異不足。新版方向改為「AI 作戰桌」:不是首頁 dashboard,而是把素材、對話、專家分派、可信來源與交付品放在同一張工作桌上。

## 競品信號

### ChatGPT Canvas

- 可學:聊天結果旁邊要有可編輯的 canvas,並支援輸出成 PDF / Word / Markdown 等交付格式。
- 對承富的啟發:AI 回答不應只停在訊息泡泡,要直接變成提案骨架、檢查表、簡報架構。
- 來源:https://help.openai.com/en/articles/9930697

### Claude Artifacts

- 可學:把對話產物變成專用視窗,支援反覆迭代、展示、分享。
- 對承富的啟發:每次投標/活動/公關任務都應該長出「可被修改的交付物」,而不是一串聊天。
- 來源:https://claude.com/blog/artifacts

### Perplexity Spaces

- 可學:每個 Space 有自己的檔案、指令、搜尋、權限與共享邊界。
- 對承富的啟發:工作包應是資料邊界,附件與對話要存在案子裡,不能散落在聊天歷史。
- 來源:https://www.perplexity.ai/help-center/en/articles/10352961-what-are-spaces

### Microsoft Copilot Pages

- 可學:AI 回答可以被送到持久 canvas,讓人和 AI 在同一頁共編。
- 對承富的啟發:老闆/企劃/設計不用進聊天找結論,而是看同一張交付頁。
- 來源:https://techcommunity.microsoft.com/blog/microsoft365copilotblog/announcing-copilot-pages-for-multiplayer-collaboration/4242701

### Notion Enterprise Search

- 可學:跨來源搜尋、研究模式、可儲存的報告、來源控制與權限遵循。
- 對承富的啟發:資料庫不是一頁搜尋框,而是 AI 回答的可信來源雷達。
- 來源:https://www.notion.com/product/enterprise-search

### Glean

- 可學:企業 AI 不是單一聊天框,而是 Search / Assistant / Agents / Connectors / Governance 的組合。
- 對承富的啟發:使用者看到的是一張工作圖,背後才是助手、搜尋、模型與權限。
- 來源:https://www.glean.com/

### Lindy Workflow

- 可學:流程由 trigger、steps、actions、conditions 組成,使用者不必從空白聊天重做重複任務。
- 對承富的啟發:投標監測、活動企劃、新聞稿發布應可從「下一步」升級為自動流程。
- 來源:https://docs.lindy.ai/fundamentals/lindy-101/create-agent

## 新版產品隱喻

舊隱喻:Dashboard + Sidebar + 多功能入口。

新隱喻:AI 作戰桌。

- 左側不是導覽,是「投入資料」。
- 中間不是卡片列表,是「AI 分派作戰圖」。
- 右側不是儀表,是「可信來源 + AI 分隊」。
- 下方不是功能區,是「交付路徑」。

## 必須保留的創新點

- 不讓使用者先選 Agent,由主管家路由。
- 聊天結果必須變成交付畫布。
- 每個答案都連到可信來源或附件。
- 工作包是 context boundary,不是專案列表。
- 重複任務能變自動流程。
