## 1 · B4 · 場勘 audio_note(1d)

**動機**:設計師事後重看場勘照片 · 想聽當下 PM 講的「光線從這邊來、有 3 根柱子擋」
**現況**:Feature #7 場勘只照片 + GPS · 沒語音

### 任務

1. 前端 `frontend/launcher/modules/site_survey.js`:
   - 加 「🎙 錄音 30 秒」按鈕(每張照片旁)
   - 用 MediaRecorder API · webm 格式 · 上傳前自動截 30 秒
   - 用 URL.createObjectURL 預覽(同 R24 memory pattern)
2. 後端 `routers/site_survey.py`:
   - 加 `POST /site/{survey_id}/audio` multipart upload
   - 重用 `routers/memory.py` 的 Whisper chain(`_transcribe_with_whisper`)
   - 結果存 `db.site_surveys.audio_notes[]`(該 survey 多個 audio note)
3. 加 1 個 test 用 mock Whisper 驗 audio_note 進 DB
4. 文件 `docs/RELEASE-NOTES-v1.3.md` Feature #4 章節

### 驗收
- [ ] iPhone PWA 點 🎙 跳系統權限 · 錄完上傳成功
- [ ] DB `site_surveys.audio_notes[].text` 有 STT 結果
- [ ] 照片詳情頁顯示「🎙 audio」icon · 點開可播 + 看 transcript

### 風險
- iOS Safari MediaRecorder 行為差(只支援部分 codec)→ 統一用 webm/opus · iOS 16+ 支援
- audio 30s 上限 · 防 DoS · 跟 R20 meeting upload 同 size guard
- Whisper 額外成本(每張照片 1 audio · 100 張 = NT$ 19.50 / 場)→ 加進 B2 cost monitor

## 2 · B1 · 社群排程真 Meta/LinkedIn API(2-3d · 大件)

**動機**:Feature #5 mock 不能真用 · 老闆要看到「FB 真的發出去了」
**現況**:`services/social_providers.py` 全 mock · 回 fake post_id

### 子階段

#### 2.1 · OAuth 設定基礎建設(0.5d)

1. Meta App 申請(承富老闆做 · 不在程式內):
   - https://developers.facebook.com/apps · 建 App
   - 加 Facebook Login + Instagram Basic Display + Pages 產品
   - 取 `app_id` + `app_secret`
   - 配 redirect URI:`https://ai.<chengfu>.com/api-accounting/social/oauth/callback`
2. LinkedIn App 申請(同上):
   - https://www.linkedin.com/developers
   - r_basicprofile + w_member_social scope
3. 寫 `scripts/setup-social-oauth.sh`:
   - 互動式 setup · 提示 paste app_id + app_secret
   - 寫 keychain · `chengfu-ai-meta-app-secret` / `chengfu-ai-linkedin-app-secret`
4. `scripts/start.sh` 加 export 進容器 env

#### 2.2 · OAuth 接入 endpoint(1d)

1. 新建 `routers/social_oauth.py`:
   ```python
   GET /social/oauth/start?platform=facebook  → 302 redirect to Meta auth
   GET /social/oauth/callback?code=...        → 換 access_token + 存
   POST /social/oauth/disconnect?platform=...  → 刪 token
   GET /social/oauth/status                   → admin 看誰連了哪 platform
   ```
2. token 存 `db.social_oauth_tokens`:
   - 結構:`{user_email, platform, access_token, refresh_token, expires_at, scopes}`
   - access_token 加密存(用 CREDS_KEY 同 LibreChat)
3. 加 `services/oauth_token_refresh.py`:
   - launchd cron 每 6h 跑 · refresh expiring tokens
   - 失敗 webhook 通知該同事 + admin

#### 2.3 · 真 Provider 實作(0.5-1d)

1. 改 `services/social_providers.py`:
   - 移除 mock · 真 call FB Graph API + LinkedIn API
   - FB: `POST /v18.0/me/feed?message=...&access_token=...`
   - IG: `POST /v18.0/{ig-user-id}/media` then `POST /media_publish`
   - LinkedIn: `POST /rest/posts` with `author=urn:li:person:...`
2. 失敗 retry · 同 v1.2 R22 exp backoff
3. response 真 `post_id` 存 `db.scheduled_posts.platform_post_id`
4. 加 「跳到原 PO」link 在 launcher · `https://facebook.com/{platform_post_id}`

#### 2.4 · Tests(0.5d)

加 5 個 integration tests(用 `responses` library mock HTTP):
- `test_oauth_callback_stores_encrypted_token`
- `test_oauth_token_refresh_cron`
- `test_dispatch_to_facebook_real_api_mocked`
- `test_dispatch_failure_retries_then_marks_failed`
- `test_admin_can_see_oauth_status`

### 驗收
- [ ] 同事在 launcher 連 FB · OAuth 跳轉成功 · 回來顯示「已連接」
- [ ] 排一篇貼文 → 5 分後 cron 跑 → FB 真出現 PO
- [ ] launcher 有「跳到原 PO」link
- [ ] /admin/social/oauth/status 看到 5 同事誰連了什麼
- [ ] token 6h 自動 refresh

### 風險
- **Meta App 審核 1-4 週**(scope 越多越久)→ 預先送 · 平行做 1.1
- IG Business Account 必須 link FB Page → 文件
- LinkedIn API 有 rate limit(100 calls / 24h / app)→ exp backoff
- FB Graph API 大改版頻繁 → pin v18.0 · 文件警告

### Fallback
若 Meta App 沒過 Sprint 3 末:
- B1 停在 OAuth 接入(2.1 + 2.2 完)
- 真 Provider(2.3)留 v1.4
- Mock 維持 · 至少 OAuth 流程 v1.3 可 demo
