## 2 · A4 · backup offsite(rclone B2 · 0.5d)

**動機**:火災 / 機器整台被偷 → 本地備份全沒 · RPO 變無限大
**現況**:`scripts/backup.sh` 已預留 `OFFSITE_REMOTE` env 但沒實作 rclone 推送

### 任務

1. `scripts/backup.sh` 加 offsite 段:
   ```bash
   if [[ -n "$OFFSITE_REMOTE" && -x "$(command -v rclone)" ]]; then
       echo "[backup] 推異機 $OFFSITE_REMOTE"
       rclone copy "$DAILY_DIR" "$OFFSITE_REMOTE" \
           --include "chengfu-*$DATE*" \
           --transfers 4 --checkers 8 \
           --log-file "$BACKUP_LOG"
   fi
   ```
2. `scripts/setup-rclone-b2.sh` 互動式設定:
   - 提示輸入 B2 application key + bucket name
   - 跑 `rclone config create chengfu-offsite b2 ...`
   - 寫入 keychain · `chengfu-ai-rclone-b2-key` 等
3. `docs/04-OPERATIONS.md` 加「異機備份設定」章節:
   - B2 vs S3 vs Cloudflare R2 比價(B2 便宜 $5/TB · 推薦)
   - 如何驗證:`rclone ls chengfu-offsite:chengfu-backup`
   - 還原:從 offsite 撈最新 → `scripts/dr-drill.sh`
4. `dr-drill.sh` 加 offsite fetch 模式(`--from-offsite`)

### 驗收

**現況**:nginx 直 serve 各 .js · 用 ?v=41 quirk cache bust

### 任務

1. 新建 `frontend/launcher/build.config.js`:
   ```js
   import esbuild from 'esbuild';
   await esbuild.build({
     entryPoints: ['src/app.js'],
     bundle: true,
     minify: true,
     sourcemap: true,
     format: 'esm',
     target: ['es2020'],
     outfile: 'dist/app.js',
   });
   ```
2. 把 `frontend/launcher/{app.js, modules/*.js, onboarding.js}` 全收進 entry

   ```json
       "build": "node build.config.js",

     },

   }
   ```
4. 改 `nginx/default.conf`:

5. `scripts/start.sh` PROD mode 跑 `npm --prefix frontend/launcher run build` 自動建
6. 把舊 `?v=41` 拿掉 · 換 esbuild 自動 hash filename(`app.[hash].js`)

## 4 · B5 · LibreChat conversation + PDPA 整合(1d · 高風險最後)

**動機**:現 PDPA delete-all 不刪 LibreChat 對話(回應有 `librechat_warning` 提醒人工)
**現況**:同事離職要 PM 跑 mongoexport 撈對話歸檔 + admin 手動 delete

### 任務

1. `routers/admin.py PDPA endpoint` 加 `include_librechat: bool = False` field
2. 新建 `services/librechat_admin.py`:
   ```python
   def find_user_id_by_email(librechat_db, email: str) -> Optional[str]:
       """LibreChat users collection 反查 _id by email · case-insensitive"""

   def archive_conversations(db, user_id: str, archive_dir: str) -> dict:
       """歸檔 user 全對話為 GPG 加密 JSON · 法務保留期內可還"""

   def delete_conversations(db, user_id: str) -> dict:
       """跨 collections 刪:conversations / messages / files / shared_links / presets"""
   ```
3. 跨 DB 連線:
   ```python
   from main import client
   librechat_db = client.LibreChat  # 同 Mongo · 不同 DB
   ```
4. PDPA endpoint 流程:
   - 找 LibreChat user_id by email(case-insensitive)
   - 若 `include_librechat=True`:
     - dry_run · 算 conv/msg/file 數
     - real · 先 archive 到 `~/chengfu-backups/offboarding/{email}-{date}/`(GPG)
     - 再刪 LibreChat 跨 5 col
   - audit 加 `librechat_archived` + `librechat_deleted` count
5. `docs/05-SECURITY.md §5.4` 改寫:
   - 不再需手動 mongoexport
   - 一個 curl 全清(含 LibreChat)
6. 加 4 個 test:
   - `test_pdpa_with_librechat_dry_run`
   - `test_pdpa_with_librechat_real_archives_first`
   - `test_pdpa_with_librechat_user_not_found`
   - `test_pdpa_archive_gpg_encrypted`

### 驗收
- [ ] dry_run · 顯示「LibreChat: 5 conversations / 234 messages / 8 files」
- [ ] real · `~/chengfu-backups/offboarding/<email>-<date>.tar.gz.gpg` 有檔
- [ ] LibreChat UI 該 user account 進不去(login 失敗)
- [ ] Mongo `db.LibreChat.conversations.count({user: <id>}) == 0`

### 風險
- LibreChat schema 改版(v0.9+)欄位名變 → pin v0.8.4 · 文件警告
- archive 前 LibreChat 仍在跑 · race(同事剛走 user 還在發訊息)→ 先 disable user 再跑 PDPA(SOP 已含 §5.2)
- GPG key 沒設 → archive 失敗 · raise 拒絕進 real delete
