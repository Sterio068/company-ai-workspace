## 1 · C3 · knowledge embedding refresh 文件(0.5d)

**問題**:knowledge_indexer 三層 timestamp 已文件(技術債#15 補)· 但 doc CHANGE 後不會自動 re-embed
**現況**:`scripts/knowledge-cron.sh` 每天跑 · 但只 incremental(用 last_search_indexed_at)
**邊角**:doc 檔案 mtime 變但 content hash 沒變 → 浪費 OCR;反之 mtime 沒變但 git rebase 改 content → 漏

**任務**:
1. 在 `services/knowledge_indexer.py` 加 SHA256 content hash 比對(覆蓋 mtime)
2. `docs/09-RAG-LAYERED-INDEX.md` 加 §5 「重建 trigger」章節
3. 加 `scripts/reembed-knowledge.py` 強制全重 embed(法務改詞 / brand-voice 大改用)
4. 1 個 unit test · `test_knowledge_indexer.py` 驗 mtime + hash 雙條件

**驗收**:knowledge cron 連跑 2 次 · 第二次只動到真改的檔(不靠 mtime trick)

## 6 · A1 · admin.py 拆檔(870 → 4 sub-routers · 1d)

**最大件 · 最後做**

**問題**:870 行(已過 800 警戒)· 含 20+ endpoint · 4 大塊邊界混亂
**現況**:單檔 含儀表 / 維運 / 內容 / 設定

**任務**:
1. 建 `routers/admin/` 目錄(從單檔變 package)

   ```

     __init__.py        ← aggregate · 給 main.py import
     dashboard.py       ← /admin/dashboard /cost /adoption /budget /top-users /tender-funnel(6 endpoint)
     data.py            ← /admin/export /import /demo-data /audit-log /ocr/reprobe(5 endpoint)
   ```

5. 全 162+ tests 必過 · 不破壞 import path

- 再一塊一塊搬 · 每搬完跑全 tests
- 最後跑 codex R33
