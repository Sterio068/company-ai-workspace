# UI Audit · 2026-04-24 · Browser Use

## 範圍

本次使用 Codex in-app Browser 驗收 `http://localhost/` 公開入口與登入前狀態。登入後 Dashboard、今日工作台、Workspace、Workflow modal、Project handoff 需要測試帳密或既有登入 session,本輪未自行建立或猜測帳號。

## 已驗證

- `http://localhost/` 在無 session 時導向 `/login`。
- 登入頁顯示公司客製中文文案:「歡迎回來」、「電子郵件地址」、「密碼」、「繼續」。
- 登入頁不再顯示「← 公司首頁」按鈕,避免未登入使用者點回首頁後被再次導回 `/login` 的循環感。
- refresh endpoint 回非 JSON 時,前端錯誤已改為可讀的 `refresh invalid response`,不再顯示 `Unexpected token` 類 JSON parse 噪音。
- 登入頁桌面窄寬畫面可讀,主要表單置中、欄位可點、底部政策連結可見。

## 已修補

- `frontend/custom/librechat-relabel.js`:登入頁不注入「回公司首頁」按鈕。
- `frontend/launcher/modules/auth.js`:refresh 成功但非 JSON response 時,丟出可讀錯誤。

## 待測

- Dashboard 今日工作台:三張卡片是否在真資料與空資料狀態都順。
- 5 Workspace:點擊是否帶入草稿、不自動送出。
- Workflow:modal、prepare-preset、saved-to-project toast、帶入主管家草稿。
- Project handoff:4 格卡、handoff prompt、atomic merge 後的會議 / 場勘資料保留。
- Mobile / PWA:場勘入口、bottom nav、相機 / 麥克風 / 定位 permission prompt。
- A11y:焦點順序、modal focus trap、reduced motion、鍵盤操作。

## 需要配合

- 提供一組測試帳號密碼,或在 in-app browser 先完成登入。
- 若要驗證 admin-only view,測試帳號需具備 `ADMIN` 或 `ADMIN_EMAILS` 身份。
